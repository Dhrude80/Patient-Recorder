# Patient-Recorder
This is a bioengineering mini project in which patient information is recorded, managed, and organized using Python. The aim of this project is to simulate a very simple patient data management system and connect programming concepts with healthcare-related record handling. This project helped me practice dictionaries, nested dictionaries, loops, conditionals, and basic data organization in Python.

Features :

Add new patient records.
View patient details.
View all stored patients.
Edit patient information.
Remove patient records.
Search for a patient using ID.
Simple and clean UI.
Fast execution.

How to Run :

Run the program.
Choose the required option from the menu.
Enter patient details such as ID, name, age, gender, blood group, and disease/condition.
Use the menu options to view, search, edit, or remove patient records.

Input :

Patient ID.
Patient name.
Age.
Gender.
Blood group.
Disease or condition.

Output :

Stored patient records.
Individual patient details.
Updated patient information after editing.
Messages for successful or invalid operations.

Concepts Used :

Python dictionaries.
Nested dictionaries.
Loops.
Conditional statements.
User input handling.
Basic record management logic.

Applications :

Basic healthcare record organization.
Understanding how patient information can be structured digitally.
Beginner practice for health-tech and bioengineering-related programming.

Requirements : Python 3.x Jupyter Notebook

AI use : Used for improving project structure, debugging logic, and understanding Python concepts
